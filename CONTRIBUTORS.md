## How to Add Yourself

Contributors are added automatically when their first PR is merged. If you've contributed and don't see your name here, please open a PR to add yourself!

### Format

```markdown
### Category
- [@username](https://github.com/username) - Description of contribution
```

### Categories

- **Project Leads** - Ongoing leadership and direction
- **Core Engineers** - Implementation of core platform features
- **AI/ML Engineers** - Model training, inference, integration
- **UI/UX Engineers** - Frontend, design, accessibility
- **Documentation** - Writing, editing, translating
- **Research** - Competitive analysis, technology evaluation
- **Design** - Architecture, API, UI/UX design
- **Community** - Moderation, support, events, outreach
- **Infrastructure** - CI/CD, release engineering, packaging
- **Security** - Audits, hardening, incident response
- **Education** - Tutorials, curriculum, workshops

## Contribution Types

We value all types of contributions:

- 💻 **Code** - Implementation, bug fixes, features
- 📖 **Documentation** - Guides, API docs, tutorials
- 🎨 **Design** - UI/UX, architecture diagrams, logos
- 🔬 **Research** - Analysis, evaluations, comparisons
- 🐛 **Bug Reports** - Issues, regressions, security
- 💡 **Ideas** - Feature requests, RFCs, discussions
- 🤝 **Community** - Support, mentoring, events
- 🌐 **Translation** - Localization, i18n
- ✅ **Testing** - Test cases, QA, benchmarking
- 📦 **Packaging** - Distribution, installers, containers
